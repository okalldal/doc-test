## Syfte

Ge plattformens aktörer ett enhetligt, spårbart och regelverksanpassat sätt att initiera, handlägga och avsluta ärenden — oberoende av vilket underliggande ärendehanteringssystem som används.

## Intressenter

| Intressent | Roll | Primärt intresse |
|---|---|---|
| Handläggare | Primär användare | Enkelt gränssnitt, korrekt statusbild |
| Medborgare / företag | Ärendeinitiator | Transparens, snabb återkoppling |
| Juridisk funktion | Regelverksansvarig | Arkivering, offentlighetsprincipen |
| Systemägare för ärendesystem | Leverantör | Väldefinierat integrationskontrakt |

## Informationsobjekt

- **Ärende** — den centrala entiteten; rymmer metadata, status och historik
- **Ärendehandling** — bifogat dokument eller strukturerad inlaga
- **Beslut** — formellt utfall av ärendehandläggning
- **Statusändring** — händelse som markerar övergång i ärendets livscykel
- **Part** — aktör med formell roll i ärendet (sökande, svarande, ombud)

## Ärendets livscykel

```
[Inkommet] ──▶ [Registrerat] ──▶ [Under handläggning]
                                        │
                        ┌───────────────┼───────────────┐
                        ▼               ▼               ▼
                  [Komplettering  [Remitterat]    [Avslutat]
                    begärd]            │               │
                        │             ▼               ▼
                        └──────▶ [Beslut fattat] ──▶ [Arkiverat]
```

## Kvalitetskrav

| Krav | Mätvärde | Målnivå |
|---|---|---|
| Spårbarhet | Andel statusändringar med loggad aktör + tidstämpel | 100 % |
| Tillgänglighet | Handläggarportalens drifttid | ≥ 99,5 % |
| Svarstid, statussök | p95 vid sökning på ärendenummer | < 500 ms |
| Arkiveringstid | Tid från avslutat ärende till arkivpost | ≤ 24 h |
| Laglig grund | Andel ärenden med registrerad rättslig grund | 100 % |

## Avgränsningar

- Förmågan hanterar **inte** den faktiska handläggningslogiken — den ansvarar för enhetlig tillgång till och status på ärenden.
- Domänspecifika ärendetyper (t.ex. bygglov, socialärenden) realiseras i respektive fackystem och exponeras via plattformens API.

## Beroenden

| Beror på | Typ | Anmärkning |
|---|---|---|
| Identitets- och åtkomsttjänst (IAM) | Hård | Autentisering och behörighetsstyrning |
| F1 – Informationsutbyte | Hård | Statusnotifieringar skickas via F1 |
| Arkivtjänst | Hård | Långtidslagring av avslutade ärenden |
| E-tjänsteplattform | Mjuk | Initiering från medborgargränssnitt |
